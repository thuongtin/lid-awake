# Support

## Before Asking For Help

Please check:

- [README](README.md) for build, run, and release commands.
- [Power Model](docs/power-model.md) for expected power-management behavior.
- [Troubleshooting](docs/troubleshooting.md) for helper approval, assertions, and closed-lid issues.
- [Releasing](docs/releasing.md) for signing and notarization questions.
- [Security Policy](SECURITY.md) for private vulnerability reporting.

## Questions

For general questions, open a discussion if the public repository has discussions enabled. If discussions are not enabled, open an issue and choose the closest matching template. For maintainer contact, email `thuongtin@gmail.com`.

Include:

- macOS version.
- Mac model.
- Whether the Mac is on AC power or battery.
- Whether Lid Awake Helper is set up and approved.
- Exact command output if the question is about build, test, packaging, or release.

## Bugs

Use the bug report issue template. Include reproduction steps, expected behavior, actual behavior, macOS version, hardware model, power source, helper status, and relevant logs with private data removed.

## Feature Requests

Use the feature request issue template. Explain the workflow, why the current behavior is insufficient, and whether the request touches helper, `pmset`, signing, launch at login, or closed-lid behavior.

## Security Issues

Do not file public issues for vulnerabilities. Follow `SECURITY.md` or email `thuongtin@gmail.com`.
